Le Hub Santé

Toutes les questions d'ordre général sur le Hub Santé

Présentation du Hub

Quelle(s) technologie(s) utilise le Hub Santé ?

Le Hub Santé s'appuie sur un message-broker RabbitMQ pour permettre l'échange de données JSON ou XML entre les logiciels raccordés au Hub Santé.

Plus de détails techniques sont disponibles dans le DST .

Est-ce que le Hub Santé est une API ?

Le Hub Santé n'est pas une API mais un message-broker RabbitMQ exposé publiquement.
A la différence d'une API, les données échangées sur le Hub Santé le sont de façon asynchrone afin de limiter le couplage entre les différents systèmes.
Ainsi, l'envoi de données à un partenaire correspond - de façon simplifiée - au dépôt du message associé sur une file d'écoute que le partenaire dépile à sa convenance.

Plus de détails techniques sont disponibles dans le DST .

Peut-on s'abonner aux messages du Hub ?

Il n'est pas possible de s'abonner à des messages passant par le Hub Santé. Les données publiées sur le Hub Santé le sont à destination d'un partenaire unique et lui seul recevra ces données.
Une fois réceptionnée par le destinataire, les données sont effacées des systèmes du Hub Santé et ne seront plus disponibles.

Est-ce que le Hub Santé stocke des données ?

Les données ne sont stockées sur le Hub Santé que lors de leur transit entre l'émetteur et le destinataire. En cas de latence ou d'indisponibilité côté destinataire, les données restent récupérables jusqu'à une durée maximum de 24h.
Cette durée peut être raccourcie si l'émetteur a configuré un TTL - Time To Live - inférieur à 24h. Au-delà ou après réception, les données sont effacées des systèmes du Hub Santé.

Est-ce que le Hub Santé fait du transcodage ?

Le Hub Santé ne fait pas de transcodage. Il partage néanmoins des nomenclatures pivôts dans ses modèles de données qui permettent à tous les acteurs de parler une langue commune pour les échanges.
Le transcodage est à la charge des acteurs se raccordant au Hub Santé pour permettre la conversion entre leurs données et les formats autorisés sur le Hub Santé.

Accompagnement

Comment puis-je m'authentifier sur le Hub Santé ?

Le Hub Santé est une plateforme d'échanges entre logiciels. C'est donc au logiciel et non directement aux utilisateurs de s'identifier sur le Hub Santé. Pour cela, les certificats SSL_SERV de l'IGC Santé sont utilisés.

Plus de détails techniques sont disponibles dans le DST .

Comment obtenir un certificat IGC Santé ?

Les certificats IGC Santé permettent d'identifier les logiciels dans le cadre des échanges sur le Hub Santé.
Pour les éditeurs souhaitant raccorder leur environnement développement/usine sur l'environnement Bac-À-Sable du Hub Santé, des certificats IGC Santé TEST sont nécessaires.
Pour les SAMU et entitées de Santé souhaitant raccorder leur logiciel en production sur les environnements de Pré-Production et Production du Hub Santé, des certificats IGC Santé PROD sont nécessaires.

Dans le cadre du Hub Santé, vous trouverez les détails d'obtention des certificats sur le Portail industriel de l'ANS .
